What are the differences between relative, absolute, and fixed positioning?

6 April 2023

In CSS, there are three types of positioning: relative, absolute, and fixed. Each of these types of positioning has its own characteristics and behaviours.

Relative positioning is used to position an element relative to its normal position in the document flow. When an element is given relative positioning, it is moved from its original position, but the space it would have occupied is still reserved in the document flow. This means that other elements will still be affected by the element's original position. Relative positioning can be applied using position: relative

Absolute positioning is used to position an element relative to its nearest positioned ancestor, or the initial containing block if there is no positioned ancestor. When an element is given absolute positioning, it is taken out of the document flow and positioned according to its ancestor's position. This means that other elements are not affected by the element's position, and the space it would have occupied is collapsed. Absolute positioning can be applied using position: absolute

Fixed positioning is used to position an element relative to the viewport, or the browser window. When an element is given fixed positioning, it is taken out of the document flow and positioned according to the viewport, which means it will stay in the same position even when the page is scrolled. This makes fixed positioning useful for creating elements that are always visible, such as headers or footers. Fixed positioning can be applied using position: fixed


I'll try to explain it in simpler terms.

Okay, imagine your webpage is like a river that flows from top to bottom. You add different elements like text, images, and buttons to the river, and each one is positioned in the river according to its default position in the HTML document flow.

Now, relative positioning is like taking an element and moving it up or down the river, but leaving the space it occupied in the flow of the river. This means that other elements in the river are still affected by the original space the element occupied.

Absolute positioning is like taking an element out of the river and placing it on a different part of the page. This means that the element won't affect other elements in the river anymore because it's no longer a part of it.

Fixed positioning is like taking an element out of the river and placing it on a boat that's floating on the river. The element will always stay in the same position, even if the river is flowing and other elements are moving along with it.


Here's a summary of the differences between these types of positioning:

* Relative positioning moves an element from its original position, but the space it would have occupied is still reserved in the document flow.

* Absolute positioning takes an element out of the document flow and positions it relative to its nearest positioned ancestor.

* Fixed positioning takes an element out of the document flow and positions it relative to the viewport, which means it stays in the same position even when the page is scrolled.

* Relative and absolute positioning can be used together to create complex layouts.

* Fixed positioning is useful for creating elements that are always visible, such as headers or footers.
